Response Format

All endpoints return JSON responses with the following structure:

{
  "success": true,
  "message": "Operation Successful",
  "data": { /* endpoint-specific data */ }
}

HTTP Status Codes

  • 200 - Success (GET, PUT)
  • 201 - Created (POST)
  • 400 - Bad Request (validation error)
  • 401 - Unauthorized (missing/invalid token)
  • 403 - Forbidden (insufficient permissions)
  • 404 - Not Found
  • 500 - Server Error

Token Management

Token Storage

Tokens are stored as httpOnly cookies for security. They are automatically included in subsequent requests.

Token Expiration
  • Default: 1 hour after login
  • With "remember": 7 days
Token Validation

All protected endpoints automatically verify the token. If invalid or expired, a 401 response is returned.
